# 001 — Initial Infrastructure Setup

## Context

Fresh project scaffold with empty `main.go`, empty `justfile`, no Docker files, and no dependencies. This plan wires up the complete infrastructure baseline: Go dependencies, config loading with env validation, a `/health` endpoint, Dockerfile, Docker Compose with profiles (postgres, redis, kratos, keto), and the justfile.

## Steps

### 1. Fix `go.mod` and install dependencies

`go.mod` currently declares `go 1.25.8` which is invalid — fix to `go 1.23`.

```sh
go mod edit -go 1.23
go get github.com/gin-gonic/gin
go get github.com/lib/pq
go get github.com/jmoiron/sqlx
go get github.com/pressly/goose/v3
go get github.com/redis/go-redis/v9
go get github.com/joho/godotenv
go get github.com/caarlos0/env/v11
go get github.com/testcontainers/testcontainers-go
go mod tidy
```

`sqlc` is a CLI tool — install via `go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest`, not a module dep.

### 2. Create `internal/config/config.go`

Struct-based config using `caarlos0/env/v11`. Single `DATABASE_URL` DSN (portable across goose, sqlx, lib/pq). All Ory URLs required. `PORT` defaults to `8080`.

```go
package config

import "github.com/caarlos0/env/v11"

type Config struct {
    Port            string `env:"PORT"              envDefault:"8080"`
    DatabaseURL     string `env:"DATABASE_URL"      required:"true"`
    RedisURL        string `env:"REDIS_URL"         required:"true"`
    KratosPublicURL string `env:"KRATOS_PUBLIC_URL" required:"true"`
    KratosAdminURL  string `env:"KRATOS_ADMIN_URL"  required:"true"`
    KetoReadURL     string `env:"KETO_READ_URL"     required:"true"`
    KetoWriteURL    string `env:"KETO_WRITE_URL"    required:"true"`
}

func Load() (Config, error) {
    var cfg Config
    return cfg, env.Parse(&cfg)
}
```

### 3. Write `cmd/api/main.go`

- Load `.env` via `godotenv.Load()` — error discarded (expected absent in containers)
- Call `config.Load()`, fatal on error
- Register `GET /health` → `{"status": "ok"}`
- Start server on `cfg.Port`

### 4. Create `.env.example` and `.env`

```
PORT=8080
DATABASE_URL=postgres://postgres:postgres@localhost:5432/webinar?sslmode=disable
REDIS_URL=redis://localhost:6379
KRATOS_PUBLIC_URL=http://localhost:4433
KRATOS_ADMIN_URL=http://localhost:4434
KETO_READ_URL=http://localhost:4466
KETO_WRITE_URL=http://localhost:4467
```

`.env` has the same content and is gitignored. `.env.example` is committed.

### 5. Create `Dockerfile`

Multi-stage:
- Stage 1: `golang:1.23-alpine` — build static binary (`CGO_ENABLED=0`)
- Stage 2: `gcr.io/distroless/static-debian12` — minimal runtime, no shell

Cache `go mod download` as a separate layer before `COPY .`.

### 6. Create `docker-compose.yml`

Services and profiles:

| Service        | Profile        | Port(s)        |
|----------------|----------------|----------------|
| postgres       | infra, full    | 5432           |
| redis          | infra, full    | 6379           |
| kratos-migrate | infra, full    | —              |
| kratos         | infra, full    | 4433, 4434     |
| keto-migrate   | infra, full    | —              |
| keto           | infra, full    | 4466, 4467     |
| api            | app, full      | 8080           |

All credentials via `${VAR:-default}` syntax. Kratos mounts `./config/kratos/kratos.yml` — needs a stub file or Docker will create a directory there.

### 7. Create `justfile`

```
run         → go run ./cmd/api
build       → go build -o bin/api ./cmd/api
test *args  → go test {{args}} ./...
lint        → golangci-lint run ./...
migrate     → goose -dir migrations postgres "$DATABASE_URL" up
generate    → go generate ./...
docker-up   → docker compose --profile {{profile}} up -d  (default: infra)
docker-down → docker compose --profile {{profile}} down
```

### 8. Directory scaffolding

- `migrations/.gitkeep` — Goose migration files go here
- `sqlc/queries/` — sqlc query `.sql` files
- `sqlc/sqlc.yaml` — sqlc config pointing at `migrations/` for schema, `sqlc/queries/` for queries, output to `sqlc/generated/`
- `config/kratos/kratos.yml` — stub Kratos config (required by Docker Compose mount)

### 9. Update `.gitignore`

Add: `.env`, `bin/`, `sqlc/generated/`

## Deviations

| # | What changed | Why |
|---|-------------|-----|
| 1 | `go.mod` stayed at `go 1.25.0` instead of being fixed to `go 1.23` | The Go toolchain installed locally is 1.25.0; `go mod edit -go 1.23` was accepted but `go get` upgraded it back. No functional impact. |
| 2 | Config moved from `internal/config/` to `pkg/config/` | Config is infrastructure-level, not domain-specific — `pkg/` is the correct location for packages reusable across `cmd/`, tests, and future tools. |
| 3 | `air` added as a `go tool` directive instead of a plain `go get` | Dev tools should not be runtime deps. `go tool` pins the version in `go.mod` without polluting the import graph, and `go tool air` works without a global install. |
| 4 | `justfile` `run` uses `go tool air` instead of `go run ./cmd/api` | Hot reload was requested during implementation; air watches `*.go` files and rebuilds automatically. |
| 5 | Postgres image changed from `postgres:16-alpine` to `postgres:18-alpine` | User preference stated during implementation. |
| 6 | Postgres volume changed from `postgres_data:/var/lib/postgresql/data` to `postgres_data:/var/lib/postgresql` | User preference stated during implementation. |
| 7 | Kratos and Keto each got a dedicated database and user (`webinar_kratos`/`kratos`, `webinar_keto`/`keto`) | User requested separate databases per service for isolation. Created via `docker/postgres/init.sql` mounted at `/docker-entrypoint-initdb.d/`. |
| 8 | `depends_on` uses `condition: service_healthy` / `service_completed_successfully` instead of plain service name | Prevents migrate containers from starting before postgres is ready, and prevents the main service from starting before migrations complete. |
| 9 | Ory image versions changed from `v1.3.1`/`v0.13.0` to `v25.4.0` | The pinned versions did not exist on Docker Hub. `v25.4.0` is the current latest for both Kratos and Keto. |
| 10 | Keto requires a `config/keto/keto.yml` config file | Keto v25 no longer accepts just a `DSN` env var — it requires a config file. Added `config/keto/keto.yml` and a `namespaces/` directory; both mounted in the compose service. |
| 11 | `config/kratos/kratos.yml` URLs are hardcoded instead of using env var substitution | Kratos v25 validates YAML values as URIs at parse time, before env substitution occurs. Since internal container ports never change, hardcoding is correct. The host-side ports remain configurable via `${KRATOS_PUBLIC_PORT:-4433}:4433` in the compose file. |
| 12 | `sqlc/generated/` not added to `.gitignore` | User decision: generated code is committed so CI can validate it without running `sqlc generate` in the pipeline. |
